## 核心难点分析

你提到的场景本质是**"可见标识（QR）→ 不可见标识（材料）"的ID继承问题**，加上**多实例并行操作**的复杂性：

| 难点 | 技术挑战 | 风险 |
|------|---------|------|
| **ID断裂** | QR码在箱体上，开箱后材料无标识 | 无法关联材料与原始二维码 |
| **多箱并行** | 用户同时搬3-4箱，逐一操作 | 箱体间遮挡、取料时来源混淆 |
| **外观一致性** | 材料卡片视觉特征高度相似 | ReID失效，ID切换（ID Switch）严重 |
| **手物交互** | 手遮挡材料、推送过程中形状变化 | 跟踪丢失、碎片化 |
| **状态判定** | 需精确识别"开箱/取料/入轨"动作节点 | 时序错乱导致数据错误 |

## 算法架构：四层级联跟踪系统

我建议采用 **"箱体-手部-材料-轨道" 四层时空关联架构**：

### 第1层：箱体与身份初始化（QR关联层）
- **检测模型**：YOLOv8-seg（实例分割，处理箱体堆叠遮挡）
- **QR识别**：ZBar + OpenCV QRDetector（双保险），识别**箱体ID**（记为 $B_i$）
- **状态跟踪**：为每个箱体维护有限状态机（FSM）
  - `ARRIVED`（到达平台）→ `OPENING`（正在开箱）→ `EXTRACTING`（取料中）→ `EMPTY`（已空）

### 第2层：手部关键点与动作识别（交互检测层）
- **检测**：MediaPipe Hands（21个关键点）或 YOLOv11-pose-hand
- **动作分类**（基于时序关键点）：
  - **开箱动作**：手部在箱体顶部，有撕扯轨迹（轨迹方向向上/向外）
  - **取料动作**：手伸入箱体，抓取后抬出（关键点与箱体IoU变化）
  - **推送动作**：手部与材料接触，水平位移 > 阈值，时间 > 1秒

### 第3层：材料卡片跟踪（核心跟踪层）
这是系统的**关键难点**。由于外观一致，必须采用**"继承式跟踪"**而非"检测式重识别"：

- **检测**：YOLOv8-small（轻量，高频30fps），检测材料卡片（记为 $M_j$）
- **跟踪器**：改进的 **ByteTrack + 自定义关联矩阵**
  - 不使用外观ReID（因为外观一致）
  - 改用**运动模型**（Kalman Filter预测）+ **手-物空间关联**

**关键创新：手-物绑定机制（Hand-Object Binding）**
```python
# 伪代码逻辑
if 手与材料IoU > 阈值 且 手处于CLOSE状态:
    材料ID = 当前活跃的手所关联的箱体ID
    建立绑定关系 (Hand H_k → Material M_j inherits Box B_i)
```

### 第4层：轨道区域与入轨判定（终点层）
- **ROI定义**：预定义轨道入口多边形区域（可配置1-4号轨道）
- **入轨事件**：材料中心点进入轨道ROI + 速度方向指向轨道内 + 手已脱离（防止误触发）

## 核心算法：时空关联图（Spatio-Temporal Association Graph）

针对**"多箱并行+外观一致"**的致命难点，我设计了一个**在线关联图算法**：

### 1. 图结构定义
- **节点**：
  - $B_i$：箱体（带QR ID）
  - $H_k$：手部实例（k=1,2，假设双手操作）
  - $M_j$：材料卡片（虚拟ID，待关联）
  - $R_m$：轨道区域（m=1,2,3,4）
- **边权重**：
  - $w(B_i, H_k)$：手与箱体的时空邻近度（最近5秒内，手在箱体内的帧数占比）
  - $w(H_k, M_j)$：手-物接触置信度（基于IoU+手部姿态 openness）
  - $w(M_j, R_m)$：材料向轨道移动的概率（基于轨迹预测）

### 2. 动态ID继承算法（关键）
当检测到**新的材料卡片** $M_{new}$ 出现时：
1. 查找当前时刻**活跃的手部** $H_{active}$（处于CLOSE或GRASP姿态）
2. 追溯该手部**最近5秒内最常交互的箱体** $B_{source}$（基于空间IoU历史）
3. 将 $B_{source}$ 的QR ID 赋给 $M_{new}$，建立**硬关联**
4. 在后续跟踪中，即使手离开，$M_j$ 仍保持该ID，直到入轨或丢失

### 3. 多箱并行消解策略
用户同时搬运4箱，如何确定材料来自哪一箱？
- **空间独占性假设**：假设用户**单手只从一个箱取料**（符合人体工学）
- **最近邻匹配**：取料瞬间，计算手与所有箱体的3D距离（如已知相机内参）或2D中心点距离，选择最近箱体
- **时序锁**：一旦手进入某箱体ROI并执行取料动作，该手被**锁定**到该箱体，直到手离开箱体区域 > 2秒

## 状态机与业务逻辑

整个系统需要维护一个**全局状态机**来确保时序正确：

```
状态定义：
- IDLE: 等待箱体
- BOX_DETECTED: 检测到新箱体（QR已识别）
- OPENING: 检测到开箱动作（箱体状态变更）
- EXTRACTING: 正在取料（手-物关联建立）
- TRANSPORTING: 材料在平台上平移（跟踪中）
- INJECTING: 推入轨道（入轨判定中）
- COMPLETED: 入轨完成，记录数据

状态转换约束：
- 必须经历了 EXTRACTING 的物料，才能进入 INJECTING（防止轨道上的残留物误触发）
- 同一材料必须按序经历各状态，不允许跳变
```

**数据记录格式**：
```json
{
  "track_id": "uuid_001",
  "qr_code": "BOX20240320A",
  "source_box": "box_instance_03",
  "extraction_time": "2026-03-20T10:23:15Z",
  "transport_path": [[x1,y1], [x2,y2], ...],  // 推送轨迹
  "target_track": "Track_2",
  "injection_time": "2026-03-20T10:23:28Z",
  "confidence": 0.94
}
```

## 技术实现建议

### 相机配置
- **位置**：俯视45°角（兼顾QR码平面和侧面取料动作）
- **分辨率**：≥1080p（QR码识别需要清晰度）
- **帧率**：30fps（推送动作跟踪需要流畅性）

### 模型选型
| 组件 | 推荐方案 | 理由 |
|------|---------|------|
| 箱体检测 | YOLOv8n-seg | Nano版够快，分割解决堆叠 |
| QR识别 | BoofCV（Java）或 ZBar | 工业级识别率，支持畸变 |
| 手部检测 | MediaPipe Hands | 21关键点，适合姿态识别 |
| 材料检测 | YOLOv8n | 小目标优化，高频推理 |
| 跟踪 | ByteTrack + 自定义关联 | 去重ID能力强，易修改 |

### 边缘计算优化（结合你之前的记忆）
根据你之前提到的边缘计算架构（本地1-4路摄像头，可切换7B-32B多模态模型）：

- **常规流程**：使用上述轻量级Pipeline（YOLO+MediaPipe），单路CPU/GPU可跑4路视频
- **Fallback机制**：当**手物关联置信度持续<0.7**超过3秒（可能遮挡严重或多手混乱），触发**7B多模态模型**（如VILA或LLaVA-Video）进行**单帧逻辑推理**：
  - 输入：当前帧+历史3帧
  - 提示词："分析当前操作：1) 哪只手正在从哪个箱子取料？2) 材料卡片当前与哪只手绑定？"
  - 输出：修正关联图的权重

## 风险与兜底方案

1. **ID漂移（最危险）**：如果跟踪器把材料A的ID错给材料B（外观一致导致），入轨记录就错了。
   - **兜底**：在**入轨前最后1秒**，增加**强制校验逻辑**：检查该ID的材料是否确实从对应箱体取出（回看该ID的出生帧的手-箱关联记录）。如果不一致，标记为"疑似错误"，人工复核。

2. **QR码遮挡**：箱体堆叠时底部箱体QR被遮挡。
   - **兜底**：要求用户**将箱体平铺**（不堆叠）在平台特定区域（画框），或允许用户**手动输入/选择**箱体ID作为备选。

3. **多手操作**：双手同时从两个箱取料（极少见但可能）。
   - **兜底**：系统检测到两个活跃手部且距离>阈值时，进入**保守模式**：暂停自动关联，提示用户"请单箱操作"或手动确认材料来源。
